# LED Message Pillar Ordering Setup

This project now has a customer order page, protected seller admin page, Supabase schema, Supabase Edge Functions, Stripe Checkout integration, and private artwork storage.

## Files Added Or Changed

- `index.html` links the main calls to action to `order.html`.
- `order.html` is the customer customization and checkout page.
- `order-confirmation.html` shows the order summary after Stripe redirects back.
- `admin.html` is the protected seller dashboard.
- `ordering.css` styles the order, confirmation, and admin pages.
- `js/order-config.js` stores the browser-side option data used for the live preview and estimate.
- `js/order.js` renders the customizer, creates the preview SVG, validates the form, and calls the checkout Edge Function.
- `js/confirmation.js` retrieves the safe order summary by Stripe Checkout Session ID.
- `js/admin.js` handles Supabase auth, order search/filtering, seller updates, signed artwork downloads, build-sheet printing, and refund/status notes.
- `js/supabase-config.example.js` is the browser config template, and `js/supabase-config.js` is the editable local browser config currently filled with placeholders.
- `.env.example` lists every required server and browser environment variable.
- `supabase/migrations/202607160001_ordering_system.sql` creates the database tables, RLS policies, private storage buckets, indexes, triggers, and starter customization option rows.
- `supabase/functions/_shared/order-config.ts` mirrors the option and price data on the server.
- `supabase/functions/create-checkout-session/index.ts` validates the order, uploads artwork/preview files, creates the Supabase order, recalculates pricing, and creates Stripe Checkout.
- `supabase/functions/stripe-webhook/index.ts` verifies Stripe signatures, handles payment events, prevents duplicate webhook processing, updates order statuses, and optionally sends confirmation email.
- `supabase/functions/order-confirmation/index.ts` returns a safe customer-facing order summary.

## Database Migration

Run `supabase/migrations/202607160001_ordering_system.sql` in your Supabase project. It creates:

- `orders`
- `order_items`
- `customization_options`
- `uploaded_artwork`
- `order_status_history`
- `webhook_events`
- `seller_profiles`

It also creates private Supabase Storage buckets:

- `order-artwork`
- `order-previews`

The `orders.order_number` field is generated as `LP-1047`, `LP-1048`, and so on by the `order_number_seq` sequence.
`orders.checkout_request_id` prevents duplicate checkout creation if the customer retries or double-clicks during the same order attempt.

## Create The Supabase Project

1. Create a new Supabase project.
2. Open SQL Editor and run `supabase/migrations/202607160001_ordering_system.sql`.
3. In Authentication, create a seller user with email/password.
4. Add that user to `seller_profiles`:

```sql
insert into public.seller_profiles (user_id, role)
values ('PASTE_AUTH_USER_ID_HERE', 'admin');
```

5. Fill in `SUPABASE_URL`, `SUPABASE_ANON_KEY`, and `FUNCTIONS_BASE_URL` inside `js/supabase-config.js`.

## Stripe Products And Webhook

The checkout function creates a dynamic Stripe line item with `price_data`, so you do not need to create a fixed Stripe Price for every customization. In Stripe, create one Product named `Custom LED Message Pillar` for reporting clarity if you want, but the final checkout amount is still generated by the server from the order options.

Create a webhook endpoint pointing to:

```text
https://YOUR_PROJECT_REF.functions.supabase.co/stripe-webhook
```

Listen for these events:

- `checkout.session.completed`
- `checkout.session.expired`
- `checkout.session.async_payment_succeeded`
- `checkout.session.async_payment_failed`
- `payment_intent.payment_failed`
- `charge.refunded`

Copy the webhook signing secret into `STRIPE_WEBHOOK_SECRET`.

## Required Environment Variables

Browser values in `js/supabase-config.js`:

- `SUPABASE_URL`
- `SUPABASE_ANON_KEY`
- `FUNCTIONS_BASE_URL`

Edge Function secrets:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `STRIPE_SECRET_KEY`
- `STRIPE_WEBHOOK_SECRET`
- `SITE_URL`

Optional email:

- `RESEND_API_KEY`
- `FROM_EMAIL`

Never put `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, or `SUPABASE_SERVICE_ROLE_KEY` in frontend JavaScript.

## Deploy Supabase Edge Functions

After logging in with the Supabase CLI and linking the project:

```bash
supabase functions deploy create-checkout-session
supabase functions deploy stripe-webhook
supabase functions deploy order-confirmation
supabase secrets set SUPABASE_URL=...
supabase secrets set SUPABASE_SERVICE_ROLE_KEY=...
supabase secrets set STRIPE_SECRET_KEY=...
supabase secrets set STRIPE_WEBHOOK_SECRET=...
supabase secrets set SITE_URL=https://yourdomain.com
```

Deploy the static website files to any static host. Set `SITE_URL` to that deployed origin, for example `https://yourdomain.com`.

## Test With Stripe Test Mode

1. Use a Stripe test secret key in `STRIPE_SECRET_KEY`.
2. Use the Stripe CLI to forward events:

```bash
stripe listen --forward-to https://YOUR_PROJECT_REF.functions.supabase.co/stripe-webhook
```

3. Put the returned `whsec_...` value into `STRIPE_WEBHOOK_SECRET`.
4. Submit `order.html` with several option combinations and a small test artwork file.
5. Pay with Stripe test card `4242 4242 4242 4242`, any future expiry, any CVC.
6. Confirm the order starts as `awaiting_payment`, then becomes `paid` only after the webhook runs.
7. Re-send the same webhook event to verify `webhook_events.stripe_event_id` prevents duplicate processing.
8. Test checkout cancel by returning from Stripe without paying.
9. Test failed payment with Stripe test card `4000 0000 0000 0002`.
10. Sign into `admin.html`, filter/search orders, download artwork, update production status, print a build sheet, and record a refund note.

## Change Options And Prices Later

Update both files when changing live prices:

- Browser preview estimate: `js/order-config.js`
- Secure server pricing: `supabase/functions/_shared/order-config.ts`

Then redeploy `create-checkout-session` so Stripe uses the new server-side prices. Optionally update `customization_options` in Supabase so the database remains a clean record of your active product configuration.
